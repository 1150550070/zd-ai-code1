【技术栈规范】
1. 核心框架: Vue 3 (Composition API, `<script setup>` 语法), TypeScript, Vite

4. 网络请求: Axios

【API 契约约束 (极其重要)】
后端的接口契约已严格按照 Schema 中的 `tableName` (转为短横线或驼峰) 统一生成。对于每一个实体，你必须使用 axios 生成对应的 API 请求方法：
假设 tableName 为 `user_info`，对应的基础路径为 `/api/userInfo`：
- 分页查询: `GET /api/userInfo/page` (参数: current, size, 以及各个条件)
- 详情查询: `GET /api/userInfo/get/{id}`
- 新增数据: `POST /api/userInfo/add` (请求体为实体 JSON)
- 更新数据: `PUT /api/userInfo/update` (请求体为实体 JSON)
- 删除数据: `DELETE /api/userInfo/delete/{id}`

【架构与目录规范】
请以标准的 Vue 3 + Vite 工程结构生成代码：
- /package.json (包含必要的依赖)
- /vite.config.ts (配置基础代理，将 /api 代理到 http://localhost:8080)
- /src/utils/request.ts (封装 Axios 实例，处理请求和响应拦截器)
- /src/api/xxx.ts (根据 Schema 的 entities，每个表生成一个 API 接口文件)
- /src/views/xxx/Index.vue (每个实体生成一个标准的增删改查表格页面，包含搜索表单、数据表格、分页、新增/编辑弹窗)
- /src/router/index.ts (配置所有的页面路由)
- /src/App.vue 和 /src/main.ts (入口文件)
