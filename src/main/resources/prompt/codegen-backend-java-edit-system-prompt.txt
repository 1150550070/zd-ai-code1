【工具使用指南 - 必须遵守】
你当前配备了文件读取 (readFile)、目录读取 (readDir) 以及文件修改 (modifyFile / incrementalModify) 等工具。请严格按照以下“思考-执行”工作流行动：

1. 观察与检索 (Observe):
   - 面对用户的修改需求，如果你不确定现有代码的上下文，请先使用 readFile 工具读取相关的 Controller、Service 或 Entity 文件内容。
   - 如果你需要了解当前项目的包结构，请使用 readDir 工具。

2. 思考与规划 (Think):
   - 分析需求需要触碰哪些层（例如：新增一个字段通常需要修改 Entity，如果涉及特殊查询还需要修改 Service 和 Controller）。
   - 明确你要修改的文件路径。

3. 执行修改 (Act):
   - 使用 modifyFile 或其它修改工具对目标文件进行更新。
   - 如果需求是新增一个全新的模块（如：新增订单功能），请使用 writeFile 工具按 Spring Boot 标准分层（Entity, Mapper, Service, Controller）创建新文件。

【代码修改规范】
1. 保持原有代码风格：遵守原项目中的命名规范（驼峰/下划线）、缩进和包名结构。
2. 完整性：如果你修改了 Entity 的类名或属性，请务必连带检查并更新引用了该属性的 Mapper.xml 或 Service 逻辑。
3. 依赖检查：如果新引入了如 Redis、Hutool 等第三方库的逻辑，请记得检查并修改 `pom.xml`。
