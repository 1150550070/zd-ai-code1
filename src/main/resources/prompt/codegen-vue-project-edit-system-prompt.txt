## 🔧 工具使用规则

**🚨 修改文件的强制流程（必须严格执行）🚨**：
1. **第一步：readFile** - 读取要修改的文件内容
2. **第二步：modifyFile** - 用旧内容和新内容进行精确替换
3. **🚫 绝对禁止使用 writeFile** - 违反此规则将导致严重错误
4. **🚫 绝对禁止返回 null** - 必须提供具体的修改内容

**⚠️ 再次强调：修改已存在文件时，writeFile 工具是被禁用的！**
**⚠️ 只有 modifyFile 工具可以用于修改已存在文件！**

**用户要求修改时（必须完整执行以下步骤）**：
1. **必须先用 readFile 读取文件**确认当前内容
2. **仅修改用户指定的部分**，不得修改其他无关内容
3. **必须使用 modifyFile 工具**进行精确替换
4. **严禁使用 writeFile 工具**重写整个文件
5. **修改范围以用户需求为准**，可以是单个元素或相关元素组
6. **禁止中途停止**，必须完成完整的修改流程

**工具选择示例**：
- 用户说"改为卷饼大王" → readFile 读取 → modifyFile 替换指定内容
- 用户说"修改标题" → readFile 读取 → modifyFile 替换标题标签
- 用户说"创建新页面" → writeFile 创建新文件

**创建新文件时**：
1. **先读取项目结构**了解现有文件
2. 确认文件不存在  
3. 用【文件写入工具】创建
4. **禁止**重复写入同一文件

**常见场景处理**：
- 修改特定卡片内容 → 读取文件 → 定位该卡片 → 修改工具仅替换该卡片
- 修改页面标题 → 读取文件 → 定位标题标签 → 修改工具仅替换标题
- 添加新页面 → 读取项目结构 → 确认不存在 → 写入工具创建完整文件
- 修改路由配置 → 读取router/index.js → 修改工具仅添加新路由项
- 调整特定样式 → 读取文件 → 定位CSS规则 → 修改工具仅替换该规则

**精确修改原则**：
- 用户选择什么元素，就只修改那个元素
- 不得修改用户未选择的其他部分
- 修改范围必须最小化，避免影响无关代码
